Introducción

¡Aprendé las respuestas a todas tus preguntas en el tutorial interactivo! O leé desde el comienzo:

GUN es un sistema de sincronización y almacenamiento de datos pequeño, fácil y rápido que se ejecuta en cualquier lugar con Javascript. El objetivo de GUN es que te centrés en los datos que necesitás almacenar, cargar y compartir en tu aplicación sin preocuparte con respecto a los servidores, las llamadas de red, bases de datos o en el seguimiento de los cambios fuera de línea o conflictos de concurrencia. Esto te permite crear aplicaciones interesantes rápidamente, como:

Offline-First

Cuando un navegador nodo solicita datos, fusionará la respuesta con sus propios datos utilizando una CRDT y luego almacenará en caché el resultado. Esto significa que:

  • La próxima vez que el navegador solicite los datos, la respuesta es instantánea, incluso sin conexión.
  • Los datos se replican en cada navegador.
  • Si el servidor falla, se pueden recuperar los datos desde los navegadores.

Esto hace que la pérdida de información importante sea casi imposible, ya que tendrían que ser destruidas todas las copias de los datos antes.

Distribuido

GUN es completamente descentralizado (nodo a nodo, de pares, peer-to-peer o multi-maestro), lo que significa que los cambios no son controlados por un servidor centralizado. Un servidor puede ser simplemente otro nodo en la red, que puede tener recursos más fiables que un navegador. Guardá los datos en una máquina y se sincronizará con otros nodos sin necesidad de un protocolo de consenso complejo. Simplemente funciona.

Pero, ¿cómo funciona?

Si te sentís perdido en palabras de moda, la siguiente descripción podría darte una idea general acerca de lo que está pasando.

La base de datos orientada a grafos (BDOG) GUN se almacena en todos los nodos que participan en la red. La mayoría de escenarios de distribución de datos que son posiles que ocurran: cada nod podría poseer el grafo completo o solamente un subconjunto y podría poseer datos que no existen en cualquier otro nodo (todavía). Se considera que toda la base de datos es la unión de los grafos de cada uno de los nodos.

No hay límite teórico para el tamaño total de un grafo de GUN. La cantidad de datos que un nodo tiene disponible localmente está limitada por las restricciones de memoria del entorno anfitrión, como el sistema operativo, el navegador, etc. La cantidad de datos que pueden persistir más del proceso que se ejecuta depende del motor de almacenamiento. En un navegador web por lo general será de 5 MB de almacenamiento local. Un supernodo con el almacenamiento de archivos Radix puede persistir cantidades de datos mucho más grandes.

Los supernodos son nodos de GUN dedicados ejecutándose en NodeJS. Ellos recibirán todos los datos que se transmiten a la red y son capaces de servir a quien lo solicite. Los nodos normales que se ejecutan en un navegador web no empezarán a descargar toda la base de datos sino solamente las partes que requieran. Una aplicación GUN bien diseñada solamente descargará los datos relevantes, en lugar de consumir el ancho de banda y la RAM de los usuarios. Esto y la falta de fiabilidad general de los navegadores nodo hacen que sea esencial tener uno o más supernodos ejecutándose 24/7 para las operaciones de la mayoría de los casos de uso del mundo real.

Instalación

GUN se puede utilizar en diversos navegadores y servidores. Hicimos que sea fácil de instalar en muchos ambientes diferentes.

Browser

Hay un par de opciones:

Etiqueta script

Lo más fácil es simplemente agregar GUN en tu HTML:

<script type="text/javascript" src="https://cdn.jsdelivr.net/npm/gun/gun.js"></script>
<script>
    // tu código acá
<script>

Require

Suponiendo que estás usando algo como Webpack o Browserify, primero seguí el proceso de instalación con npm, luego agregá esto al código del navegador:

var Gun = require('gun/gun');

Import

Igual que con el require, pero utilizando la sintaxis ES6:

import Gun from 'gun/gun';

Nota: Por el momento, a pesar de que se importa, GUN todavía se define y utiliza como una variable global.

Node

Primero necesitás instalar GUN with NPM o Yarn a través de la línea de comandos:

$npm install gun o $yarn add gun

Nota: Si no tenés instalado node o npm, leé esto.

Después agregá esto al código del servidor:

var Gun = require('gun');

Nota: GUN viene con muchos adaptadores NodeJS por defecto para el almacenamiento y redes. Si no los querés, solamente usa require('gun/gun') en su lugar.

Server

Si querés instalar GUN en un servidor es necesario pasar una instancia de HTTP:

var server = require('http').createServer().listen(8080);
var gun = Gun({web: server});

Te recomendamos que mirés el ejemplo de HTTP(S) por defecto, o el de Express y Hapi que están en la misma carpeta.

Hola Mundo

Creemos una sencilla aplicación Hola Mundo en un pequeño archivo que contendrá tanto el código HTML y JS:

<html>
  <body>
    <h1>Hola</h1>
    <p>¿Cómo te llamás?</p>
    <form><input><button>Enviar</button></form>

    <script src="https://cdn.jsdelivr.net/npm/gun/examples/jquery.js"></script>
    <script src="https://cdn.jsdelivr.net/npm/gun/gun.js"></script>
    <script>
      var gun = Gun()

      $('form').on('submit', function (event) {
        event.preventDefault()
        var input = $('form').find('input')
        gun.get('hola').put({ nombre: input.val() });
        input.val('')
      })

      gun.get('hola').on(function(data, key) {
        var h1 = $('h1')
        h1.text('Hola ' + data.nombre)
      })
    </script>
  </body>
</html>

Probalo, pero introduciendo tu nombre. El mensaje de saludo lo mostrará.

Ahora refrescá la página entera. Tu nombre todavía estará allí gracias a GUN.

¿Qué está pasando?

Primero cargamos GUN con <script src="https://cdn.jsdelivr.net/npm/gun/gun.js"></script> y lo inicializamos con var gun = Gun();

Luego, cada vez que el usuario envía un nuevo nombre $('form').on actualizaremos el nodo GUN con un objeto { nombre: input.val() } hacemos put en la clave 'hola' con gun.get('hola').put(...).

Finalmente, usando gun.get('hola').on(...) obtenemos actualizaciones en tiempo real de cada vez que el nodo 'hola' cambia, entonces podemos mostralo en el h1.

Tené en cuenta que cuando se actualice la página, se llamará automáticamente a gun.get('hola').on(...), porque GUN sabe que ya existen datos en el nodo 'hola'
